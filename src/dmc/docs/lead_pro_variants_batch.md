create a new signature.

## Key Findings for DMC Parser Development

1. **The Morbid/Onslaught "Pro" variants (V4 side: 4.0 Prof → 4.X → 4.y, V5 side:
   5.0+ → 5.1x → 5.1y) all credit Brian**, suggesting semi-official collaboration.
   The player format likely matches standard V4/V5 with additions. Songs from these
   editors will have SIDID signatures matching "DMC V4.0 Pro" or "DMC V5 Pro" variants.

2. **The independent modifications** (Xlcus paulk/XL, Rayden/Alpha Flight, Warlock/Panic,
   Stryyker/Tide, Zeor/Samar, TAT) may have different player signatures. The Double
   SID version (Rayden) is structurally different — it manages two SID chips and needs
   a specialized packer.

3. **DMC V4.3++ (Stryyker/Tide)** has no CSDb entry but IS in HVSC and has its own
   SIDID signature. The HVMEC page confirms it uses standard V4 key bindings, so the
   editor UX is unchanged. The "++" in the version suggests player enhancements.

4. **DMC 5.1y is the most complete Pro V5 release**: highest downloads (1,161), dedicated
   loader (Explorer), documentation (DJB + The Syndrom), The Syndrom's charset. Songs
   from 5.1y may use features not in standard V5.1.

5. **Brian's triple-group attribution** (Graffity + The Imperium Arts + Tomcat) appears
   in DMC V4.2 credits, suggesting the Tomcat-era code (earliest Brian work) was
   acknowledged in the credit chain even for 1990s releases.

6. **No technical documentation** exists on CSDb for any of these variants — all pages
   lack description text. The only actionable technical information comes from:
   - Disassembling the binaries (D64/ZIP downloads)
   - HVMEC pages (brief notes only)
   - SIDID signatures (player detection)
   - Cross-referencing with the standard V4/V5 player analysis
