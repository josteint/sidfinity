Rob Hubbard Driver Research — Source Index
============================================
Compiled: 2026-04-11

FETCHED AND DOCUMENTED
=======================

1. Cadaver's Music Programming Essay
   URL: https://cadaver.github.io/rants/music.html
   File: cadaver_music_essay.txt
   Content: Complete technical essay on building a C64 music routine. Key details
   about hard restart (old vs testbit method), logarithmic vibrato (Hubbard's
   technique), pulse width modulation, waveform/arpeggio tables, pattern formats
   (Hubbard, SadoTracker, GoatTracker), sequence data, optimization strategies.

2. VGMPF Wiki — Rob Hubbard (C64 Driver)
   URL: https://www.vgmpf.com/Wiki/index.php?title=Rob_Hubbard_(C64_Driver)
   File: vgmpf_hubbard_driver.txt
   Content: Driver evolution timeline from 1984-1987. Feature additions per game.
   Table-based drums/PWM dates. 4-bit PCM info. Composers who used the driver.

3. C64Audio.com "Master of Magic" Blog
   URL: https://c64audio.com/blogs/rob-hubbard-master-of-magic
   File: c64audio_master_of_magic.txt
   Content: Four blog posts — primarily historical/biographical. Key technical
   insight: classic driver ended with Hollywood or Bust, new driver from April 1986.
   NOTE: The actual book contains 6502 assembly driver excerpts not available online.

4. Classic Driver Format (Monty on the Run disassembly)
   URL: https://www.1xn.org/text/C64/rob_hubbards_music.txt
   Also: https://codebase64.org/doku.php?id=magazines:chacking5 (C=Hacking #5)
   File: hubbard_classic_driver_format.txt
   Content: MOST VALUABLE SOURCE. Complete byte-level format: 8-byte instruments,
   variable-length pattern encoding, track/sequence hierarchy, vibrato algorithm,
   portamento, drum implementation, gate management, pulse width, execution flow.

5. SID History and Community Knowledge
   URLs: Multiple (atlantis-prophecy.org, akaobi.wordpress.com, c64-wiki.com,
          lemon64.com forums, chipmusic.org)
   File: sid_history_and_techniques.txt
   Content: Hubbard's data approach (raw SID values vs Galway's conversion),
   player efficiency, composition workflow, channel multiplexing, sample playback,
   community editor development, analysis tools.

6. New Driver (1986-1987) — Known Facts
   File: hubbard_new_driver_1986.txt
   Content: What we know about the post-classic driver: table-based drums/PWM,
   filter support, PCM playback, Delta's custom code path. Gaps identified for
   reverse engineering. Recommended analysis approach.

7. SIDID Signatures
   File: sidid_hubbard_signatures.txt
   Content: Exact byte patterns from tools/sidid.cfg for identifying Hubbard
   drivers. Analysis of each pattern. Confirms 8-byte instrument format and
   $FF/$FE pattern terminators.

KEY TOOLS FOR FURTHER REVERSE ENGINEERING
==========================================
1. SIDdecompiler (https://github.com/Galfodo/SIDdecompiler)
   Originally "HubbardDecompiler" — produces labeled assembly for Hubbard tunes.

2. player-id (https://github.com/WilfredC64/player-id)
   Identifies Hubbard driver variants by byte signatures.

3. Our own sidxray toolkit — can trace register accesses and detect data regions.

4. siddump --writelog — cycle-accurate register dumps.
